# 같이 먹자 — 프로젝트 지침

## UI 변경 검증 방침

가벼운 UI 변경(아이콘 교체, 색상/여백/문구 조정, 스타일 값 수정 등)은 코드 리뷰만으로 충분하다.
Claude가 직접 브라우저를 띄워서 스크린샷으로 확인할 필요 없음 — 코드가 올바르게 참조·적용됐는지 정적으로 확인하는 선에서 마무리한다.

브라우저로 실제 확인이 필요한 경우:
- 인터랙션 로직이 바뀌는 변경 (클릭 시 상태 전이, 애니메이션 타이밍, 조건부 렌더링 분기 등)
- 레이아웃이 크게 바뀌어 다른 요소와 겹치거나 깨질 가능성이 있는 변경
- 사용자가 명시적으로 "확인해줘" / "테스트해줘"라고 요청한 경우

참고: 이 앱은 로그인(Google/카카오/이메일)이 필요해 Claude가 가진 브라우저 세션으로는 로그인 이후 화면까지 도달하기 어렵다. 아이콘 등 시각 요소만 빠르게 확인하고 싶으면 실제 화면 대신 위젯(mcp__visualize)으로 SVG를 렌더링해서 보여주는 방식을 우선 사용한다.
